Building our Login Page

Chances are, your users will need to login; let's build an easy one using the template that we have!


Updating the Nav

As we did in the last section, we'll need to update our navigation first. Hopefully you'll know how to handle this by now! I've attached a screenshot of what it should look like below just in case you forgot.



Inserting Our Logo

Hopefully you remember the beautiful logo you made earlier in this tutorial! If you already have one, great, otherwise please backtrack here!

To get us kicked off, pop your logo into your image folder (should be img -> logo.png/jpg/etc).

Here's the easy part, since we're building a login page we won't need cards for the styling. All you have to do is pop in your logo with a standard img tag. Don't forget to make sure it's responsive too by inputting style: "width: 100%". I also went ahead and animated it too for some extra flair by appending on class = "w3-animate-top" on top. Psst, if you still need help with what animation to use, check this W3 tutorial out.



Making a Login Form

W3.CSS provides a plethora of options to choose from when it comes to building a login page or any form in general. You can check it out here!

We'll start by building the structure of the form itself with form class = "w3-container">. Depending on what you're using to pull/push data to/from the back end you may want to include a form action as well. Since this tutorial is front-end based though, we won't go into that stuff.



For this project, we'll have a login comprised of two basic inputs: a username and a password. The user will need to input both in order to get through, so let's make these validated! Luckily, W3.CSS offers a nifty solution to that using their input class styling. We're going to copy and paste their validation form into our own code like so. I used the one from here.



Let's wrap things up with some nice looking buttons! We'll need two for this: a LOGIN and a New User option. To start, since we don't want our buttons to collide with one another, and they aren't necessarily a completely a different card/entity from the form we will break them up with the w3-section class.

Within these sections we'll need to make our actual buttons. These full width buttons should do the trick. In order to differentiate them, we can adjust their sizing with the sizing classes inherently included in w3. For our LOGIN let's pop on the w3-xlarge class and for our New User let's use the w3-medium class. I also hope you remember the accent color you chose way back, as we'll be using that to give our buttons life on the screen. I've added a w3-light-green class to both buttons to do so.
